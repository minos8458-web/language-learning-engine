# PINYIN_NORMALIZATION_STRESS_TEST.md
## Pinyin Normalization Stress Test — 다음자(多音字) 10종 검증

> 새 Language Pack을 추가하지 않는다. IDENTIFIER_STANDARD §3(v1.9)의 병음 정규화 규칙이 Grammar·Vocabulary·Identifier 세 레이어에서 실제로 버티는지, 대표적인 다음자 10종으로 스트레스 테스트한다.

---

## 0. 문서의 지위

- 새 Language Pack이 아니라 **기존 Experimental 규칙 하나를 집중 검증하는 보고서**다.
- 대상: `了`(기존 1건, 재확인용 베이스라인) + 신규 10종 = 총 11개 한자.

---

## 1. 핵심 발견 — 두 종류의 다음자가 있다

이번 테스트로 다음자 문제가 **성격이 다른 두 유형**으로 나뉜다는 것이 드러났다.

| 유형 | 정의 | 이번 세트에서의 비율 |
|---|---|---|
| **A. 비충돌형** | 두 발음이 성조 제거 후에도 서로 다른 문자열이 됨(음절 자체가 다름) | 7/11 (64%) |
| **B. 충돌형** | 두 발음이 성조만 다르고 로마자 음절은 동일해, 성조 제거 시 SLUG가 같아짐 | 3/11 (27%) |

**B유형이 27%라는 것이 이번 테스트의 핵심 결론이다** — 중국어에서 성조 충돌은 드문 예외가 아니라 **일상적으로 마주치는 빈도**다. Pinyin Normalization 규칙은 "충돌이 어쩌다 발생하면 그때 대응"하는 수준이 아니라 처음부터 충돌 해소 절차를 포함하고 있어야 했다는 뜻이며, 다행히 **그 절차가 이미 다른 이유로 존재하고 있었다**(4장).

---

## 2. 개별 검증 결과

### 2.1 유형 A — 비충돌형 (7종)

| 한자 | 발음 1 | 발음 2 | SLUG 1 | SLUG 2 | 문법 관련성 |
|---|---|---|---|---|---|
| 了 | le(완료상) | liǎo(이해하다) | `LE` | `LIAO` | `GRAMMAR_ZH_LE` 기존 검증(VALIDATION_REPORT_ZH_v1.0) |
| 行 | xíng(가능/걷다) | háng(줄/업종) | `XING` | `HANG` | 둘 다 순수 어휘 — Grammar Node 불필요, Vocabulary 전용 사례 |
| 长 | cháng(길다) | zhǎng(자라다/우두머리) | `CHANG` | `ZHANG` | 어휘 형용사/동사, Grammar Node 불필요 |
| 乐 | lè(즐겁다) | yuè(음악) | `LE` | `YUE` | 어휘. **`lè`의 SLUG(`LE`)가 기존 `GRAMMAR_ZH_LE`(了)와 문자열은 같지만 엔터티 타입 접두사가 달라(`VOCAB_` vs `GRAMMAR_`) 실제 충돌 아님** — 3.2 참고 |
| 着 | zhe(지속상) | zháo(닿다) / zhuó(입다) | `ZHE` | `ZHAO`/`ZHUO` | `zhe`는 지속상 표지로 v1.1 Grammar Node 후보(ASPECT, 4.6 참고) |
| 重 | zhòng(무겁다) | chóng(다시) | `ZHONG` | `CHONG` | 어휘, Grammar Node 불필요 |
| 还 | hái(아직/또한) | huán(돌려주다) | `HAI` | `HUAN` | **기존 `GRAMMAR_ZH_HAI_MEI`가 정확히 `hái` 발음을 썼음을 소급 확인** |

### 2.2 유형 B — 충돌형 (3종, 핵심 발견)

| 한자 | 발음 1 | 발음 2 | 공통 SLUG | 품사 차이로 해소 가능한가 |
|---|---|---|---|---|
| 得 | dé(얻다, VERB) | de(정도보어 조사, PARTICLE) | `DE` | **가능** — VOCAB_ZH_DE_V / VOCAB_ZH_DE_PART |
| 发 | fā(보내다/발생하다, VERB) | fà(머리카락, NOUN) | `FA` | **가능** — VOCAB_ZH_FA_V / VOCAB_ZH_FA_N |
| 为 | wéi(~이다, VERB) | wèi(~때문에, PREPOSITION) | `WEI` | **가능** — VOCAB_ZH_WEI_V / VOCAB_ZH_WEI_PREP |

(참고: `得`의 세 번째 발음 `děi`(필요, 이미 `GRAMMAR_ZH_DEI`로 존재)는 음절 자체가 `dei`로 달라 `de`/`dé`와 충돌하지 않는다.)

## 3. 레이어별 검증 결과

### 3.1 Grammar Layer

- 11개 한자 중 실제로 Grammar Node가 필요한 것은 `了`(이미 존재)와 `着`(zhe, v1.1 후보)뿐이다. 나머지는 전부 순수 어휘라 Grammar Node를 만들지 않는 것이 맞는 판단이었다 — **"다음자라고 전부 문법 노드가 되는 것은 아니다"**라는 새로운 확인 사항이다.

### 3.2 Vocabulary Layer

- 유형 A 7종은 VOCABULARY_SCHEMA §3(Lemma/Surface Form 분리)만으로 문제없이 등재된다.
- 유형 B 3종은 IDENTIFIER_STANDARD §9(Vocabulary ID)에 **이미 존재하던** 품사 기반 DISAMBIGUATOR 규칙(`VOCAB_EN_LIE_V`/`VOCAB_EN_LIE_N` 선례)을 그대로 재사용해 전부 해소됐다. **새 규칙을 만들 필요가 없었다.**
- `乐`(lè)와 `了`(le)의 SLUG 문자열 자체는 같지만(`LE`), 전자는 Vocabulary Entry(`VOCAB_ZH_LE`), 후자는 Grammar Node(`GRAMMAR_ZH_LE`)라 엔터티 타입 접두사가 이미 이 둘을 구분한다 — 이는 IDENTIFIER_STANDARD §11(공통 형식 규칙, "모든 ID는 엔터티 타입 접두사로 시작")이 의도치 않게 다음자 문제의 절반을 이미 막고 있었다는 뜻이다.

### 3.3 Identifier Layer

- IDENTIFIER_STANDARD §3(v1.9)의 3단계 절차(문법 기능 결정→Vocabulary에서 발음 확정→Identifier 생성)가 11개 사례 전부에서 문제없이 작동했다.
- **새로 필요했던 규칙은 0건이다.** 충돌 해소는 §9의 기존 DISAMBIGUATOR가, 크로스 엔터티 충돌 방지는 §11의 접두사 규칙이 이미 담당하고 있었다.

---

## 4. 잔여 위험 (완전히 해소되지 않은 것)

**같은 SLUG, 같은 품사인 충돌 사례는 이번 세트에 없었다.** 이번 3건의 충돌은 전부 품사가 달라 기존 규칙으로 풀렸지만, 만약 같은 품사·같은 SLUG인 두 발음이 발견되면(예: 두 동사가 우연히 성조만 다르고 로마자·품사가 같은 경우) 현재 규칙으로는 구분할 수 없다. 이런 사례는 이번 10종에서 발견되지 않았으므로 **이론적 잔여 위험으로만 기록**하고, 실제로 발견되면 그때 세 번째 층위의 DISAMBIGUATOR(예: 의미 태그 또는 성조 번호)를 검토한다.

## 5. 결론 — 승격 판단

**Pinyin Normalization 규칙을 Experimental에서 승격할 것을 권고한다.**

| 판단 근거 | 내용 |
|---|---|
| 사례 수 | 11개(비충돌 8 + 충돌 3), 언어 특정 규칙의 승격 기준(2~3건)을 크게 초과 |
| 새 메커니즘 필요 여부 | **0건** — 모든 사례가 이미 존재하던 규칙(§9 품사 DISAMBIGUATOR, §11 접두사 규칙)만으로 해소됨 |
| 잔여 위험 | 이론적으로 남아 있으나(4장) 실제 발견 사례 없음, 발견 시 대응 절차(3단계 DISAMBIGUATOR)까지 이미 설계 방향이 잡혀 있음 |

**등급 제안**: 언어 특정 규칙은 원래 정의상 "여러 언어" 기준의 Frozen이 적용되지 않으므로, "이 언어 안에서는 규칙이 완전히 안정됐다"는 의미로 **`Frozen(언어 특정)`**이라는 하위 등급을 신설해 부여할 것을 제안한다 — 다른 언어 기준의 `Frozen`(3개 언어 검증)과 구분되는 표기다.

이 판단은 LANGUAGE_VALIDATION_SUMMARY_V1.md §10 갱신과 함께 확정한다.
